Resumen&#58; 
 Una QR Web, será un sitio web que se distribuirá a partir de un código QR, y cuyos visitantes podrán consultar el impacto de una “cua_user”, o de “una tipología a de pods dentro de una cua_user particular” (por ejemplo el impacto de “ALMACENES EXITO” de la cua_user “exito”). &#160; 
&#160; 
Para realizar esta implementación se contarán de los siguientes aspectos&#58; 

 URL &#58; diseño de la URL para que a partir de la misma se realicen las consultas correspondientes y se muestre la información de impacto correspondiente. 

 Menú superior principal &#58; vinculará a secciones definidas dentro del sitio web principal de Eatcloud (https&#58;//eatcloud.com) 

 Cabezote principal &#58; constará de una imagen, el nombre de la cua_user , el nombre de la tipología_a de pods, un slogan, y un llamado a la acción. &#160;La persistencia en donde se guardará la información del cabezote principal, será utilizada también para establecer si se genera o no una QR_WEB (active=true), de tal manera que solamente cuentas usuario que estén en dicho repositorio y que estén activas, se les desplegará la QR_WEB. 

 Nuestro impacto en números &#58; inicialmente mostrará cuatro indicadores de impacto, pero en el futuro se podrán presentar más. 

 Nuestros programas de impacto social &#58; sección en donde por cada cua_user se presentarán programas de impacto social (en caso de existir dichos programas, es decir, en caso de que el contenido sobre esos programas exista. &#160;Si el contenido no existe, entonces la sección no se muestra) 

 Historias de impacto &#58; sección en donde por cada cua_user se presentarán historias de impacto social (en caso de existir dichas historias, es decir, en caso de que el contenido sobre esas historias exista. &#160;Si el contenido no existe, entonces la sección no se muestra). 

 Valor acumulado &#58;&#160;presentará una sumatoria del valor generado por todos los programas sociales y el valor al costo de las donaciones realizadas. 
&#160; 
A continuación se presenta un diseño preliminar, sobre el cual se podrán realizar ajustes para adecuarse mejor a los requisitos. 
&#160; 
Esta página deberá estar internacionalizada, de tal manera que pueda ser &#160;consultada en diversos idiomas.
