Resumen: 
 Dadas las proyecciones de nuevas funcionalidades para EatCloud, que implicarán la creación de nuevas “nubes” o “cuentas maestras” para un mismo territorio, se deberá verificar que el acceso de un usuario a una “nube”, se pueda establecer por un parámetro de configuración, de tal manera que diversos usuarios, de la misma APP, puedan ingresar a nubes o donaciones de diversas cuentas maestras.  En principio, en la APP un usuario podrá (inicialmente) ingresar a una sola “nube” (cua_master). 

 Aproximación inicial: cuenta maestra: parámetro del usuario 
 En la última reunión de “nuevas funcionalidades” se estableció que el ingreso a la APP modernizada, a diferencia de lo que ocurría en legacy, no implica la consulta de una coordenada para establecer la cuenta maestra del país y luego ingresar a la nube correspondiente, dado que el usuario estaría asociado directamente a una cuenta maestra.  En principio este principio debería funcionar para habilitarle diferentes nubes a diferentes usuarios, pero se debe validar esta afirmación.  Para ello se crearán diversas cuentas maestras para Colombia y se deberá validar en la APP modernizada, que funcione de manera adecuada para diferentes usuarios.
