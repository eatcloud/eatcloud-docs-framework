Resumen&#58; 
 En la actualidad, cuando se crea una donación en la WAPP se cuenta con la funcionalidad de adjuntar archivos. &#160;En este momento dicha funcionalidad no permite adjuntar .xml. 
&#160; 
 Se solicitó que desde la creación de la donación, y también desde funcionalidades de edición de la misma, se puedan adjuntar archivos .xml, que contienen Facturas Electrónicas, como un complemento a la funcionalidad de certificación con este tipo de soporte. &#160; 
&#160; 
 En ese orden de ideas se debe coordinar muy bien la subida de este tipo de documento, para que esté disponible en las funcionalidades de generación de certificado que lo requieren y establecer si esta subida tendrá o no validaciones con respecto a la donación que se crea o edita. 
&#160; 
 Definición de mecanismos de integración con la funcionalidad de certificados de donación 
Se deberán establecer, endpoints, rutas o carpetas en las cuales se deberán subir los .xml para que estén disponibles para la funcionalidad de certificados de donación. &#160;De igual manera se debe establecer si desde la WAPP se realizarán validaciones propias del proceso de “subida de facturas” por el “BO Donante (datagov_cuentas)”. &#160;Una vez se cuente con estas definiciones, estas serán insumos para la implementación de la subida de .xml por la WAPP. 
&#160; 
 Subida de .xml asociados a una donación 
Una vez definidos cómo debe operar la subida de este tipo de archivos por la WAPP, se deberán operar para que de manera similar a cuando se sube un adjunto a la donación, esta información esté disponible para el proceso de certificados de donación.
