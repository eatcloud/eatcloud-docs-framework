Descripición general 
Cuando se creó la nueva cuenta “alimentoscarnicos”, por defecto el campo “delete_donation” se creó con valor “true”. &#160;Esto hizo que usuarios inescrupulosos de la WAPP, pudieran borrar desde dicha plataforma donaciones. &#160; 
&#160; 
Para evitar futuros olvidos, se debe configurar la base de datos, de tal manera que el valor por defecto del campo “delete_donation” cuando se cree un nuevo registro (una nueva cua_user) sea “false” (es decir, que por defecto se le impidan borrar donaciones desde la WAPP).
